# Wasm Tools Monorepo

A high-performance Monorepo containing WebAssembly (Wasm) utility libraries built in Rust. This repository compiles core logic into dual-purpose libraries designed to run efficiently in browser environments and inside a local Node.js CLI tool for seamless debugging and automation.

## 📂 Project Structure

This project is managed as a Monorepo using Rust Workspaces and Node.js Workspaces:

```text
├── crates/                 # 🦀 Rust Core Source Code (Multiple independent libraries)
│   ├── library_a/          # e.g., Image processing core
│   └── library_b/          # e.g., Cryptography core
├── packages/               # 📦 Compiled JS Glue-code & CLI tool
│   ├── lib-a/              # JS wrapper generated by wasm-pack for library_a
│   ├── lib-b/              # JS wrapper generated by wasm-pack for library_b
│   └── cli/                # 🟢 Node.js CLI wrapper for local debugging
├── Cargo.toml              # Rust Workspace configuration
├── package.json            # Node.js Workspaces configuration
└── Makefile                # Build and orchestration tasks
```

## 🛠️ Prerequisites

Ensure you have the following tools installed locally before developing:

- **Rust**: Latest stable toolchain (`rustup`)
- **wasm-pack**: Wasm compiler for Rust (`cargo install wasm-pack`)
- **Node.js**: Version 18 or higher

## 🚀 Getting Started

All development workflows—including dependency installation, multi-crate compilation, testing, and CLI execution—are encapsulated inside the top-level `package.json` scripts and the `Makefile`. 

Please refer to the `Makefile` targets or the `"scripts"` section in the root `package.json` for a full list of commands to build and run the project locally.
